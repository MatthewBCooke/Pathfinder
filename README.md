<h2><a href="#synopsis" aria-hidden="true" class="anchor" id="user-content-synopsis"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Synopsis</h2>
<p>The jslssa package (Jason Snyder Lab Search Strategy Analysis) is a animal search strategy analysis tool for the Morris Water Maze. The program analyses X-Y coordinate data exported from tracking software (currently only supports Ethovision). We then calculate the best-fit search strategy for the trial. We analyse the data and fit it into one of 9 search strategies: Direct Swim, Directed Search, Focal Search, Spatial indirect, Chaining, Scanning, Thigmotaxis, and Random Search.</p>
<h2><a href="#usage-example" aria-hidden="true" class="anchor" id="user-content-usage-example"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Usage Example</h2>
<ol>
<li>
<p>The program can be opened by calling <code>jslSearch</code>. See <a href="https://github.com/Norton50/JSL/blob/master/README.md#installation"><strong>Installation</strong></a> for install instructions.</p>
</li>
<li>
<p>This will open up the GUI window.</p>
</li>
</ol>
<p><a href="https://camo.githubusercontent.com/950047053156e6633fd5758a5a8a26f61b27bf16/68747470733a2f2f692e6779617a6f2e636f6d2f38363965393666313966656531326439346434343235333536303766383834652e706e67" target="_blank"><img src="https://camo.githubusercontent.com/950047053156e6633fd5758a5a8a26f61b27bf16/68747470733a2f2f692e6779617a6f2e636f6d2f38363965393666313966656531326439346434343235333536303766383834652e706e67" alt="Window Preview" data-canonical-src="https://i.gyazo.com/869e96f19fee12d94d442535607f884e.png" style="max-width:100%;"></a></p>
<ol start="3">
<li>
<p>You can then select an inividual file or a directory containing files from the <strong>File</strong> dropdown menu. These files must be Excel files if you are using Ethovision tracking software, and CSV files if you are using Anymaze or Watermaze software. <em>Note: For directory selection, you must enter <strong>into</strong> the directory and not just highlight it for it to be selected</em></p>
</li>
<li>
<p>From here you can choose to either <a href="https://github.com/Norton50/JSL/blob/master/README.md#a-generating-heatmaps">(<strong>A</strong>)</a> generate heatmaps for the chosen trials, or to <a href="https://github.com/Norton50/JSL/blob/master/README.md#b-search-strategy-analysis">(<strong>B</strong>)</a> calculate search strategies.</p>
</li>
</ol>
<h3><a href="#a-generating-heatmaps" aria-hidden="true" class="anchor" id="user-content-a-generating-heatmaps"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>(A) Generating Heatmaps</h3>
<p>The jslssa package allows for the efficient generation of heatmaps. To do so, follow these steps.</p>
<ol>
<li>
<p>Click on <strong>File</strong> -&gt; <strong>Generate Heatmaps</strong></p>
</li>
<li>
<p>A parameters panel will appear:</p>
</li>
</ol>
<p><a href="https://camo.githubusercontent.com/7d9fd3bb54c50a2562fa37e883708c1e694cd235/68747470733a2f2f692e6779617a6f2e636f6d2f65346466636238373037383433336632373664373835343331616633626639362e706e67" target="_blank"><img src="https://camo.githubusercontent.com/7d9fd3bb54c50a2562fa37e883708c1e694cd235/68747470733a2f2f692e6779617a6f2e636f6d2f65346466636238373037383433336632373664373835343331616633626639362e706e67" alt="Heatmap parameters" data-canonical-src="https://i.gyazo.com/e4dfcb87078433f276d785431af3bf96.png" style="max-width:100%;"></a></p>
<ol start="3">
<li>
<p>The parameters panel lets you tailor the output to your needs:</p>
<ol>
<li>
<p>Grid size. This roughly translates into how many bins to put the data in. For more information on grid size see matplotlib documentation: <a href="http://matplotlib.org/devdocs/api/_as_gen/matplotlib.axes.Axes.hexbin.html">http://matplotlib.org/devdocs/api/_as_gen/matplotlib.axes.Axes.hexbin.html</a></p>
</li>
<li>
<p>Maximum Value. This will allow you to change at which value the points in the heatmap will become their most saturated (dark red). Setting 'Auto' will dynamically assign the maximum value to be equal to the value of the maximum grid.</p>
</li>
<li>
<p>Dynamic Parameters. Our dynamic parameter display will appear if you are using Ethovision. This display allows you to tick the boxes of which labels you would like to consider. Example: <code>(X) Genotype | WT</code> will only calculate heatmaps from trials with WT genotype animals.</p>
</li>
</ol>
</li>
<li>
<p>You can then click generate, and our software will plot a heatmap of your trial data.</p>
</li>
</ol>
<p><a href="https://camo.githubusercontent.com/b201250bc8184ac9e3635f855f21e66926fd2fa8/68747470733a2f2f692e6779617a6f2e636f6d2f33383231316464396138616464393037666661366239353561646633303538652e706e67" target="_blank"><img src="https://camo.githubusercontent.com/b201250bc8184ac9e3635f855f21e66926fd2fa8/68747470733a2f2f692e6779617a6f2e636f6d2f33383231316464396138616464393037666661366239353561646633303538652e706e67" alt="heatmap display" data-canonical-src="https://i.gyazo.com/38211dd9a8add907ffa6b955adf3058e.png" style="max-width:100%;"></a></p>
<h3><a href="#b-search-strategy-analysis" aria-hidden="true" class="anchor" id="user-content-b-search-strategy-analysis"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>(B) Search Strategy Analysis</h3>
<ol>
<li>
<p>For search strategy analysis we have multiple options. To use predefined values from either Snyder et el., 2017, Ruediger et al., 2012, or Gathe et al., 2009 click their respecive buttons. To set your own strategy parameters, click custom.</p>
</li>
<li>
<p>(Custom) The custom button will spawn a parameters panel</p>
</li>
</ol>
<p><a href="https://camo.githubusercontent.com/17cfe25016ce7cc262026f3568e42bf0c6a31647/68747470733a2f2f692e6779617a6f2e636f6d2f66376133343532326234643462636162316263313734366330323138373637622e706e67" target="_blank"><img src="https://camo.githubusercontent.com/17cfe25016ce7cc262026f3568e42bf0c6a31647/68747470733a2f2f692e6779617a6f2e636f6d2f66376133343532326234643462636162316263313734366330323138373637622e706e67" alt="custom parameters" data-canonical-src="https://i.gyazo.com/f7a34522b4d4bcab1bc1746c0218767b.png" style="max-width:100%;"></a></p>
<ol start="3">
<li>
<p>In the custom parameters pane, you can select and deselect any of the search strategies. Deselecting Strategies will remove them from consideration. You can also define the cutoff values for each strategy. For definitions of these values see Snyder et al., 2017.</p>
</li>
<li>
<p>Once you have chosen your parameters, be sure to select your tracking software. Ethovision, Anymaze, and Watermaze are currently supported. (<em>Note: Anymaze and Watermaze are currently in beta -- not all features are available</em>)</p>
</li>
<li>
<p>You may then alter the main values to suit your data. Platform position, pool centre, and pool diameter can be automatically calculated for groups of non-probe trials tracked in Ethovision. For all other data you must manually define these values (Example: <code>Platform Position (x,y) | 6.53,-17.3</code>). Old platform position is only used when Perseverance is chosen in the Custom parameters pane. For more in-depth explanations of these values, see Snyder et al., 2017.</p>
</li>
<li>
<p>There are 3 checkboxes above the <strong>Calculate</strong> button. The first, <em>Scale Values</em> is used to automatically scale the default values in an attempt to better match your data. This uses the Pixels/cm and the pool diameter to determine a constant C with which to multiply some parameters. (<em>Note: If you are using custom values, it is best to disable scaling</em>) The two other checkboxes enable manual categorization. Manual categorization can be used for trials in which our algorithm was unable to make a determination (<strong>Manual categorization for uncategorized trials</strong>) or for all trials (<strong>Manual categorization for all trials</strong>).</p>
</li>
</ol>
<p><a href="https://camo.githubusercontent.com/3cb4bb5c77d270eadf3c8bccf7fbb5c0f5876c78/68747470733a2f2f692e6779617a6f2e636f6d2f61333934623466313865363933636232366639393663636132376665616331362e706e67" target="_blank"><img src="https://camo.githubusercontent.com/3cb4bb5c77d270eadf3c8bccf7fbb5c0f5876c78/68747470733a2f2f692e6779617a6f2e636f6d2f61333934623466313865363933636232366639393663636132376665616331362e706e67" alt="manual categorization" data-canonical-src="https://i.gyazo.com/a394b4f18e693cb26f996cca27feac16.png" style="max-width:100%;"></a></p>
<ol start="7">
<li>Once you are satisfied with your parameters, click calculate. This will begin the process of determining search strategies for the trials. Once calculation is complete you will be shown a display of the results.</li>
</ol>
<p><a href="https://camo.githubusercontent.com/783d8ce7a96944f445967989c2981a97d2efbd6b/68747470733a2f2f692e6779617a6f2e636f6d2f64336339313866323737383564336138623931306664383730666431383338372e706e67" target="_blank"><img src="https://camo.githubusercontent.com/783d8ce7a96944f445967989c2981a97d2efbd6b/68747470733a2f2f692e6779617a6f2e636f6d2f64336339313866323737383564336138623931306664383730666431383338372e706e67" alt="csv display" data-canonical-src="https://i.gyazo.com/d3c918f27785d3a8b910fd870fd18387.png" style="max-width:100%;"></a></p>
<ol start="8">
<li>Your results will be saved as a <code>.csv</code> file with whatever name was chosen in the <em>Output File</em> field. You will also receive a log file of the excecution, and any generated paths saved in your present working directory.</li>
</ol>
<h2><a href="#motivation" aria-hidden="true" class="anchor" id="user-content-motivation"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Motivation</h2>
<p>This program was developed in order to simplify as well as remove inconsistencies in Morris Water Maze search strategy analysis.</p>
<h2><a href="#installation" aria-hidden="true" class="anchor" id="user-content-installation"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation</h2>
<p>This package is hosted on PyPi, and is therefore easy to install with PIP.
To install simply call: <code>pip install jslssa</code></p>
